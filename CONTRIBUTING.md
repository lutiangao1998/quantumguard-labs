# Contributing to QuantumGuard Labs

First off, thank you for considering contributing to QuantumGuard Labs! It's people like you that make the open source community such a great place. We welcome any form of contribution, from reporting a bug to submitting a feature request, from writing documentation to implementing new features.

## Code of Conduct

This project and everyone participating in it is governed by the [QuantumGuard Labs Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to [contact@quantumguard.io](mailto:contact@quantumguard.io).

## How Can I Contribute?

### Reporting Bugs

- **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/lutiangao1998/quantumguard-labs/issues).
- If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/lutiangao1998/quantumguard-labs/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** or an **executable test case** demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

- Open a new issue to discuss your enhancement. Clearly state the **problem** you are trying to solve and the **proposed solution**.
- Provide as much context as possible so that we can understand the full scope of the enhancement.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`pytest`).
5. Make sure your code lints.
6. Issue that pull request!

## Styleguides

### Git Commit Messages

- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.
- Reference issues and pull requests liberally after the first line.

### Python Styleguide

- We follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) with a line length of 88 characters.
- We use `black` and `isort` to automatically format our code.

We look forward to your contributions!
