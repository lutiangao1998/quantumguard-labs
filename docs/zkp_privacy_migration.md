# ZKP 隐私迁移 (ZKP Privacy Migration) 技术文档

## 1. 简介

ZKP 隐私迁移功能旨在通过零知识证明（Zero-Knowledge Proofs）技术，在资产从量子脆弱地址迁移到抗量子安全地址的过程中，切断链上的直接关联，从而保护用户的交易隐私。

## 2. 核心组件

### 2.1 ZKPMigrationProver
负责在客户端生成 ZK 证明。它接收源地址、目标地址、金额和秘密随机数（Nullifier），并输出一个不包含源地址信息的证明。

### 2.2 ZKPMigrationVerifier
负责在服务端或链上验证证明的有效性。只有验证通过的隐私迁移请求才会被执行。

## 3. 工作流程

1.  **初始化**：用户选择要迁移的资产和目标 PQC 地址。
2.  **证明生成**：客户端生成一个 ZK 证明，证明其拥有源地址资产的控制权，并承诺将其发送到目标地址。
3.  **提交请求**：用户向 QuantumGuard Labs 提交包含 ZK 证明和公共信号（如承诺值、金额）的隐私迁移请求。
4.  **验证与执行**：系统验证 ZK 证明。验证通过后，通过隐私中继器（Relayer）或混合器（Mixer）执行链上迁移操作。

## 4. 安全性考虑

*   **Nullifier 机制**：防止双重花费（Double-spending）攻击。
*   **可信设置 (Trusted Setup)**：对于 Groth16 等协议，需要进行安全的可信设置。
*   **PQC 签名集成**：所有的 ZK 证明提交过程均由 PQC 签名保护，防止中间人攻击。
