# Q-Swap (抗量子 DEX) 技术文档

## 1. 简介

Q-Swap 是 QuantumGuard L2 (Q-Chain) 生态中的核心去中心化交易所 (DEX)，旨在提供一个完全抗量子的资产交易平台。它采用自动化做市商 (AMM) 模型，允许用户在 Q-Chain 上安全地交易抗量子资产，并为流动性提供者 (LP) 赚取交易费用。

## 2. 核心功能

### 2.1 抗量子资产交易

Q-Swap 支持 Q-Chain 上所有 PQC 安全资产的交易对，包括 QG 代币、Q-BTC、Q-ETH 和 Q-Stablecoin 等。所有交易都将强制使用 Q-Chain 原生 PQC 签名进行验证，确保交易过程的抗量子安全性。

### 2.2 流动性提供与激励

用户可以通过向流动性池提供两种资产来成为流动性提供者。作为回报，LP 将获得流动性提供者代币 (LP Token)，并赚取交易费用分成。

### 2.3 价格发现与滑点控制

Q-Swap 采用恒定乘积做市商模型 (x * y = k)，通过池中资产的比例自动确定价格。前端界面将提供价格滑点预警，帮助用户控制交易风险。

### 2.4 交易费用

每笔交易将收取少量费用（例如 0.3%），其中大部分将分配给流动性提供者，剩余部分可能用于协议发展或 QG 代币回购销毁。

## 3. 技术实现

### 3.1 智能合约架构

Q-Swap 的核心逻辑将部署为一系列抗量子智能合约在 Q-Chain 上，包括：
*   **Factory 合约**：用于创建新的交易对（流动性池）合约。
*   **Pair 合约**：每个交易对对应一个 Pair 合约，管理流动性池中的资产和交易逻辑。
*   **Router 合约**：提供用户友好的交易接口，支持多跳交易路径。

### 3.2 PQC 签名集成

所有与 Q-Swap 智能合约的交互（如 `swap`、`addLiquidity`、`removeLiquidity`）都将通过 `Q-SDK` 进行 PQC 签名。智能合约内部将包含 PQC 签名验证逻辑，确保只有经过 PQC 签名的交易才能被执行。

### 3.3 交易流程

1.  **用户授权**：用户首先需要授权 Q-Swap Router 合约访问其钱包中的资产。
2.  **构建交易**：用户在前端选择交易对和数量，Q-SDK 构建 PQC 签名的交易。
3.  **PQC 签名**：用户使用其 PQC 私钥对交易进行签名。
4.  **广播交易**：签名后的交易被广播到 Q-Chain 网络，由 Q-Swap 智能合约执行。

## 4. 风险管理

*   **无常损失 (Impermanent Loss)**：作为 AMM 的固有风险，将通过文档和前端提示向 LP 明确。
*   **智能合约风险**：所有合约将经过严格的第三方安全审计。
*   **PQC 算法风险**：依赖于底层 Q-Chain 的 PQC 算法安全性。

## 5. 未来展望

*   集成限价单、止损单等高级交易功能。
*   支持集中流动性 (Concentrated Liquidity) 模型，提高资本效率。
*   与其他 Q-DeFi 协议（如 Q-Lending）深度集成，提供更丰富的金融服务。
