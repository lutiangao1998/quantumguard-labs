# 原生 PQC 共识引擎 (Q-Consensus) 技术文档

## 1. 简介

Q-Consensus 是 QuantumGuard L2 (Q-Chain) 的核心，它是一个原生抗量子的共识引擎。通过将后量子密码学 (PQC) 算法深度集成到共识机制中，Q-Consensus 确保了 Q-Chain 在面对未来量子计算机攻击时，依然能够保持其区块的完整性和网络的安全性。

## 2. 核心特性

### 2.1 原生 PQC 签名

Q-Consensus 采用 NIST 标准化的 PQC 签名算法（如 ML-DSA-65 或 Falcon-512）对每个区块进行签名。这意味着即使量子计算机能够破解传统的 ECDSA 或 RSA 签名，Q-Chain 的区块依然无法被伪造或篡改。

### 2.2 灵活的共识机制

Q-Consensus 设计为可插拔的共识模块，可以根据网络需求适配不同的共识算法，例如：
*   **权益证明 (Proof-of-Stake, PoS)**：通过验证者质押 QG 代币来参与区块生产和验证。
*   **委托权益证明 (Delegated Proof-of-Stake, DPoS)**：允许代币持有者投票选举验证者。

### 2.3 验证者管理

提供一套完整的验证者注册、注销和管理机制，确保网络的去中心化和安全性。

## 3. 技术实现

### 3.1 PQC 签名集成

Q-Consensus 通过 QuantumGuard Labs 的 `AgilityLayer` 模块与底层 `liboqs` 库进行交互，实现 PQC 密钥的生成、签名和验证。每个区块的签名都将明确指定所使用的 PQC 算法。

### 3.2 区块结构

Q-Chain 的区块结构将包含以下关键字段：
*   `timestamp`：区块创建时间。
*   `data`：区块中包含的交易数据或其他 L2 状态更新。
*   `validator_id`：签名该区块的验证者 ID。
*   `pqc_algorithm`：用于签名该区块的 PQC 算法。
*   `signature`：验证者使用其 PQC 私钥对区块数据生成的签名。

### 3.3 签名与验证流程

1.  **区块创建**：验证者收集交易，构建区块数据，并使用其 PQC 私钥对区块数据进行签名。
2.  **区块广播**：签名后的区块被广播到 Q-Chain 网络。
3.  **区块验证**：其他验证者接收到区块后，使用签名验证者的 PQC 公钥验证签名的有效性。

## 4. 安全性

*   **抗量子安全性**：共识层直接采用 PQC 签名，从根本上抵御量子攻击。
*   **去中心化**：通过共识机制确保验证者的多样性和网络的抗审查性。
*   **可审计性**：所有区块签名和验证过程均可追溯和审计。

## 5. 未来展望

*   集成更先进的 PQC 算法。
*   实现更复杂的共识机制，如异步拜占庭容错 (aBFT)。
*   支持轻客户端验证。
