# Q-Stablecoin (抗量子稳定币) 技术文档

## 1. 简介

Q-Stablecoin 是 QuantumGuard L2 (Q-Chain) 生态中的抗量子稳定币协议。它旨在提供一个由 Q-Chain 上 PQC 安全资产超额抵押的稳定币，作为 Q-Chain 生态内的价值锚点和交易媒介，确保其价值稳定性和抗量子安全性。

## 2. 核心功能

### 2.1 PQC 资产超额抵押

Q-Stablecoin 的发行将由 Q-Chain 上的 PQC 安全资产（如 Q-BTC, Q-ETH, QG 代币等）进行超额抵押。这意味着每发行 1 单位的 Q-Stablecoin，都需要锁定价值超过 1 单位的 PQC 资产作为抵押品，以确保其价值稳定性。

### 2.2 铸造与销毁机制

用户可以通过锁定 PQC 资产来铸造 Q-Stablecoin，也可以通过销毁 Q-Stablecoin 来赎回其抵押品。所有铸造和销毁操作都将通过 PQC 签名进行验证。

### 2.3 价值稳定性

Q-Stablecoin 的价值将锚定主流法币（如美元），通过超额抵押和动态抵押率调整机制来维持其价格稳定。

### 2.4 抗量子安全

Q-Stablecoin 的核心机制（铸造、销毁、抵押品管理）都将强制要求 PQC 签名，从而有效抵御量子计算对传统密码学签名的威胁，保障稳定币的绝对安全。

## 3. 技术实现

### 3.1 智能合约架构

Q-Stablecoin 的核心逻辑将部署为一系列抗量子智能合约在 Q-Chain 上，包括：
*   **Vault 合约**：用于锁定用户的 PQC 抵押资产。
*   **Minting 合约**：管理 Q-Stablecoin 的铸造和销毁逻辑。
*   **Price Oracle**：提供 PQC 抵押资产的实时价格数据，用于抵押率计算和清算。

### 3.2 PQC 签名集成

所有与 Q-Stablecoin 智能合约的交互（如 `mint`、`redeem`、`depositCollateral`、`withdrawCollateral`）都将通过 `Q-SDK` 进行 PQC 签名。智能合约内部将包含 PQC 签名验证逻辑，确保只有经过 PQC 签名的操作才能被执行。

### 3.3 铸造与赎回流程

1.  **用户抵押**：用户将 PQC 资产存入 Vault 合约作为抵押品。
2.  **铸造 Q-Stablecoin**：根据抵押品价值和抵押率，用户可以铸造相应数量的 Q-Stablecoin。
3.  **赎回**：用户销毁 Q-Stablecoin，并从 Vault 合约中赎回其抵押品。

## 4. 风险管理

*   **智能合约漏洞**：所有合约将经过严格的第三方安全审计。
*   **预言机攻击**：依赖于 PQC 安全的价格预言机，防止价格操纵。
*   **抵押品风险**：抵押资产的价格波动可能导致抵押率不足，需优化清算机制。

## 5. 未来展望

*   支持更多类型的 PQC 资产作为抵押品。
*   集成自动化清算机器人，提高协议的稳定性。
*   探索算法稳定币模型，减少对超额抵押的依赖。
