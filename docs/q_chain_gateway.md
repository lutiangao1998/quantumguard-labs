# Q-Chain 资产跨链网关 (Q-Gateway) 技术文档

## 1. 简介

Q-Gateway 是 QuantumGuard L2 (Q-Chain) 与其他 Layer 1 (L1) 区块链之间进行资产安全转移的关键组件。它确保了资产在 L1 和 Q-Chain 之间进行存入 (Deposit) 和提现 (Withdrawal) 过程中，能够全程受到后量子密码学 (PQC) 的保护，抵御量子攻击。

## 2. 核心功能

### 2.1 抗量子资产存入

用户可以将 L1 上的资产（如 BTC、ETH）通过 Q-Gateway 存入 Q-Chain。存入过程中的关键交易和指令将使用 PQC 签名进行认证，确保资产来源的真实性和指令的不可篡改性。

### 2.2 抗量子资产提现

用户可以从 Q-Chain 提现资产到指定的 L1 地址。提现指令同样会通过 PQC 签名进行验证，防止恶意提现或伪造提现请求。

### 2.3 多链支持

Q-Gateway 设计为可扩展架构，未来将支持与多个主流 L1 区块链进行资产互操作，包括比特币、以太坊、Solana 等。

## 3. 技术实现

### 3.1 PQC 签名认证

Q-Gateway 深度集成了 QuantumGuard Labs 的 `AgilityLayer`。所有存入和提现请求的关键数据都会被序列化，并使用用户的 PQC 私钥进行签名。Q-Gateway 在处理这些请求时，会使用对应的 PQC 公钥进行验证。

### 3.2 存入流程 (L1 -> Q-Chain)

1.  **用户发起存入请求**：用户在 L1 上向 Q-Gateway 的智能合约或多签地址发送资产，并提供 L2 接收地址。
2.  **L1 交易确认**：Q-Gateway 监控 L1 上的存款交易，并在达到一定确认数后确认存款。
3.  **PQC 签名**：用户使用其 L1 账户的 PQC 私钥对存款指令进行签名。
4.  **Q-Chain 铸造**：Q-Gateway 在 Q-Chain 上为用户铸造等量的抗量子资产，并发送到指定的 L2 接收地址。

### 3.3 提现流程 (Q-Chain -> L1)

1.  **用户发起提现请求**：用户在 Q-Chain 上向 Q-Gateway 发送提现请求，并提供 L1 接收地址。
2.  **PQC 签名**：用户使用其 Q-Chain 账户的 PQC 私钥对提现指令进行签名。
3.  **Q-Chain 销毁**：Q-Gateway 在 Q-Chain 上销毁等量的抗量子资产。
4.  **L1 释放**：Q-Gateway 在 L1 上向用户指定的 L1 接收地址发送等量的 L1 资产。

## 4. 安全性

*   **抗量子签名**：所有关键操作均由 PQC 签名保护，防止量子计算机伪造交易。
*   **多重验证**：结合 L1 交易确认和 PQC 签名验证，确保资产转移的安全性。
*   **审计日志**：所有跨链操作都会被记录，方便审计和追踪。

## 5. 未来展望

*   支持更复杂的跨链场景，如 NFT 跨链。
*   集成去中心化预言机，增强跨链数据的可靠性。
*   实现更高效的跨链机制，如原子交换。
