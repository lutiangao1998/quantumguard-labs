# QuantumGuard Labs - 项目架构

本文档详细描述了 QuantumGuard Labs 量子安全迁移平台（QMP）的技术架构、模块划分与目录结构。

## 1. 整体架构理念

平台采用模块化、服务化的设计理念，旨在构建一个可扩展、高内聚、低耦合的系统。核心逻辑围绕"风险识别 -> 迁移规划 -> 安全执行 -> 合规审计"的闭环流程展开。架构设计充分考虑了密码敏捷性（Crypto-Agility），确保系统能够平滑过渡到未来的后量子密码（PQC）标准。

整体架构分为四个核心业务层和一个基础服务层：

- **风险分析层 (Analyzer)**：负责连接区块链网络，解析链上数据，识别并量化存量资产的量子风险敞口。
- **迁移编排层 (Orchestrator)**：基于风险分析结果和预设策略，负责制定、管理和执行资产迁移计划。
- **密码敏捷层 (Crypto)**：提供统一的密码学抽象，支持传统密码与后量子密码的混合签名模式，并采用可插拔的算法设计。
- **审计与证明层 (Auditor)**：负责生成迁移过程的可验证日志、安全证明和合规报告。
- **基础核心层 (Core)**：提供与区块链节点的连接、数据解析、交易构建等通用基础能力，为上层业务模块提供支持。

## 2. 核心模块设计

### 2.1 风险分析器 (Analyzer)

- **职责**：识别链上资产的量子攻击暴露面。
- **实现** (`bitcoin_analyzer.py`)：连接比特币节点，遍历 UTXO 集。通过分析交易脚本（ScriptPubKey），识别出公钥直接暴露（如 P2PK）或可被推断（如 P2PKH 在公钥重用后）的地址。
- **数据模型** (`models.py`)：定义风险等级（高、中、低）、资产信息（地址、金额、UTXO）和风险原因（公钥暴露类型）等数据结构。

### 2.2 迁移编排器 (Orchestrator)

- **职责**：根据策略安全、高效地执行资产迁移。
- **实现** (`migration_planner.py`, `policy_engine.py`)：接收风险清单，根据策略（如"优先迁移高价值资产"、"单批次不超过100 BTC"）生成迁移计划。支持与多签审批系统、HSM 集成，构建并广播迁移交易。

### 2.3 密码敏捷层 (Crypto)

- **职责**：解耦业务逻辑与具体密码算法，实现未来的平滑升级。
- **实现** (`agility_layer.py`)：定义统一的签名和验证接口，支持 ECDSA、Schnorr、ML-DSA（CRYSTALS-Dilithium）等算法的混合签名方案。

### 2.4 审计与证明器 (Auditor)

- **职责**：为迁移活动提供不可篡改的证据和合规报告。
- **实现** (`proof_generator.py`, `reporting.py`)：记录所有迁移操作的哈希链日志，生成"量子就绪度"报告。

## 3. 技术选型

| 技术领域 | 选型 |
|---|---|
| 后端语言 | Python 3.11+ |
| 数据分析 | Pandas, NumPy |
| 区块链交互 | python-bitcoinlib / Bitcoin Core RPC |
| API 框架 | FastAPI |
| 后量子密码 | OQS-Python (liboqs) |
| 数据库 | PostgreSQL |
| 报告生成 | ReportLab / WeasyPrint |
